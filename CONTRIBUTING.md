# Contributing to CampVerse

Welcome to CampVerse! We're excited to have you contribute to our mission of building a robust event management platform. As an open-source project under the **GNU GPLv3**, we value transparency and collaboration.

## Your Role as a Contributor

By contributing to this project, you agree that your contributions will be licensed under the same GPL-3.0 license.

## How to Contribute

### 1. Find an Issue

Before starting work, please check our issue tracker. If you have a new idea or find a bug, open a new issue first to discuss it with the maintainers.

### 2. Fork and Branch

1.  Fork the repository to your own account.
2.  Create a descriptive branch name (e.g., `fix/login-bug` or `feature/user-profiles`).

### 3. Standards

- **Code Style**: Follow the existing ESLint and Prettier configurations.
- **Testing**: Ensure any new logic is accompanied by tests (if applicable).
- **Security**: Do not commit secrets (API keys, passwords, etc.). Use environment variables.

### 4. Pull Request (PR) Process

- Submit a PR against the `main` branch.
- Provide a clear summary of your changes.
- Reference the issue your PR addresses.
- Wait for a maintainer to review and approve your code. **Only maintainers can merge code into the official repository.**

## Code of Conduct

Please be respectful and helpful to all members of the community. We aim to foster a collaborative and welcoming environment.

---

Thank you for helping us make CampVerse better!
